## 소개

**Vault DAO**는 블록체인 기술과 안전한 파일 저장소를 결합한 탈중앙화 디지털 자산 보안(DAS) 플랫폼입니다. 다음과 같은 기능을 제공합니다:

- **제로 트러스트 아키텍처**: 아무도 믿지 않고, 모든 것을 검증
- **블록체인 기반 보안**: 투명하고 변경 불가능한 거래 기록
- **클라이언트 측 암호화**: 데이터가 기기를 떠나기 전에 암호화됨
- **샤미르 비밀 공유**: 보안 강화를 위해 지갑 키를 여러 조각으로 분할
- **그룹 협업**: 팀원들과 자산 공유 및 관리

## 시작하기

### 계정 생성

#### 1단계: 로그인 페이지 접속

`http://localhost:3000`에서 애플리케이션을 처음 방문하면 로그인 페이지로 리디렉션됩니다.

![로그인 페이지](01-login-page.png)

로그인 페이지 구성:
- 이메일 입력 필드
- 비밀번호 입력 필드
- 로그인 버튼
- 회원가입 페이지 링크

#### 2단계: 회원가입으로 이동

새 계정을 만들려면 **"회원가입"** 링크를 클릭하세요.

![회원가입 페이지](02-signup-page.png)

회원가입 페이지 입력 필드:
- **이름** - 선택사항
- **이메일** - 필수*
- **비밀번호** - 필수* (최소 8자)
- **비밀번호 확인** - 필수*

#### 3단계: 정보 입력

회원가입 양식에 정보를 입력하세요:

![작성된 회원가입 양식](03-signup-filled.png)

**요구사항:**
- 이메일은 유효한 이메일 주소여야 합니다
- 비밀번호는 최소 8자 이상이어야 합니다
- 비밀번호가 일치해야 합니다

#### 4단계: 등록 제출

**"회원가입"** 버튼을 클릭하여 계정을 생성하세요.

회원가입 후 서비스 약관에 동의해야 합니다.

- **"전체 약관에 동의합니다"** 체크박스를 선택

![약관 동의 완료](05-terms-accepted.png)

모든 약관에 동의하면 **"확인"** 버튼이 활성화됩니다. 클릭하여 진행하세요.

### 지갑 설정

약관 동의 후 블록체인 지갑 설정을 진행합니다.

#### 1단계: 지갑 소개

![지갑 설정 소개](06-wallet-setup-intro.png)

소개 페이지에서 보안 기능을 설명합니다:
- **클라이언트 측 암호화** - 키가 암호화되지 않은 상태로 기기를 떠나지 않음
- **샤미르 비밀 공유 (2-of-3)** - 키가 3개 부분으로 분할되며, 2개로 복구 가능
- **PIN 보호** - 6자리 PIN이 하나의 조각을 암호화

**조각 분배:**
- **조각 A**: 브라우저에 저장 (기기에만)
- **조각 B**: PIN으로 암호화되어 서버에 저장
- **조각 C**: 서버에 저장 (핫 조각)

**"Continue"**를 클릭하여 진행하세요.

#### 2단계: PIN 설정

![지갑 PIN 설정](07-wallet-pin-setup.png)

지갑을 보호할 6자리 PIN을 선택하세요:

![PIN 입력 완료](08-wallet-pin-filled.png)

**중요 사항:**
- PIN을 기억하세요 - 지갑과 금고에 접근하는 데 필요합니다
- PIN은 지갑 키 조각 중 하나를 암호화하는 데 사용됩니다
- PIN 없이는 조각 A와 조각 C가 있어야만 지갑을 복구할 수 있습니다

**"Create Wallet"**을 클릭하여 블록체인 지갑을 생성하세요.

3. **그룹 자산** - 공유 그룹 금고
4. **그룹 관리** - 그룹 생성 및 관리
5. **나의 지갑** - 블록체인 지갑 관리
6. **활동 로그** - 모든 활동 및 거래 내역 보기

### 사용자 프로필

오른쪽 상단에 표시되는 정보:
- **멤버십 레벨**: Bronze, Silver, Gold 또는 Platinum
- **이메일 주소**: 등록된 이메일
- **스토리지 사용량**: 현재 사용량 / 총 사용 가능 용량 (예: 0MB / 10.0GB)
- **로그아웃 버튼**

### 대시보드 콘텐츠

메인 대시보드는 다음에 대한 정보를 제공합니다:
- **제로 트러스트 아키텍처**: 플랫폼의 보안 원칙
- **프라이빗 디지털 세이프티 박스 (PDX)**: 맞춤형 접근 제어 기능
- **DAS 기술**: 양자 안전 암호화 기술

## 나의 자산 - 개인 금고

**"나의 자산"**을 클릭하여 개인 파일 금고에 접근하세요.

![나의 자산 금고 잠김](10-my-assets-vault-locked.png)

### 금고 비밀번호

개인 금고는 6자리 비밀번호로 보호됩니다(지갑 PIN과 별도). 이는 저장된 파일에 대한 추가 보안 계층을 제공합니다.

**최초 설정:**
- 금고 비밀번호를 아직 설정하지 않았다면 생성해야 합니다
- **"계정"** → **"비밀번호"**로 이동하여 금고 비밀번호를 설정하세요

### 금고 열기

1. 다이얼 인터페이스에 6자리 금고 비밀번호를 입력하세요
2. **"금고열기"**를 클릭하세요
3. 파일과 폴더가 표시됩니다

**비밀번호 복구:**
- 금고 비밀번호를 잊어버린 경우 **"마이페이지/비밀번호관리"**로 이동하세요
- 안내된 복구 절차를 따르세요

## 나의 지갑 - 블록체인 지갑

**"나의 지갑"**을 클릭하여 블록체인 지갑과 디지털 자산을 관리하세요.

![나의 지갑 페이지](14-my-wallet.png)

### 지갑 개요

**표시 정보:**
- **ETH 잔액**: 현재 이더리움 잔액
- **지갑 주소**: 고유한 블록체인 주소 (예: 0x756a48633DBeA6331C830815B5ab03536c422689)
- **네트워크**: 현재 블록체인 네트워크 (개발용 Hardhat Local)
- **체인 ID**: 네트워크 식별자 (로컬 Hardhat의 경우 1337)
- **RPC URL**: 블록체인 노드 엔드포인트 (localhost:8545)

### 지갑 작업

**주소 복사 버튼**:
- 지갑 주소를 클립보드에 빠르게 복사
- 암호화폐를 받기 위해 이 주소를 사용

**탐색기에서 보기**:
- 블록체인 탐색기에서 지갑 열기
- 거래 내역과 잔액 확인

**재발급**:
- 필요한 경우 새 지갑 주소 생성
- 주의해서 사용 - 이전 지갑에서 새 지갑으로 자산 이전

**테스트 ETH 받기**:
- 개발 모드에서 10 테스트 ETH 받기
- 거래 및 기능 테스트에 사용

### 지갑 탭

나의 지갑 페이지에는 다양한 유형의 디지털 자산 관리를 위한 여러 탭이 있습니다:

#### ETH 탭

![ETH 잔액](14-my-wallet.png)

이더리움 잔액을 보여주는 기본 탭:
- **잔액 표시**: 현재 ETH 잔액 표시
- **새로고침 버튼**: 블록체인에서 잔액 업데이트
- **네트워크 정보**: 현재 네트워크, 체인 ID, RPC URL 표시
- **테스트 ETH 버튼**: 개발/테스트용 10 ETH 받기

#### 거래내역 탭

![거래 내역](21-wallet-transactions.png)

모든 블록체인 거래 확인:
- **거래 목록**: 모든 입출금 내역
- **거래 세부정보**: 금액, 날짜, 송수신 주소, 상태
- **새로고침 버튼**: 거래 내역 업데이트
- **빈 상태**: 거래가 없을 때 "거래 내역이 없습니다" 표시

#### NFT 탭

![NFT 컬렉션](22-wallet-nft.png)

대체 불가능한 토큰 관리:
- **NFT 갤러리**: NFT 컬렉션의 시각적 표시
- **NFT 발행**: 금고 자산에서 NFT 생성
- **빈 상태**: "No NFTs Yet" 메시지와 나의 자산 섹션에서 발행 안내
- **NFT 세부정보**: 메타데이터, 소유권 및 이력 확인

#### STO 탭 (증권형 토큰)

![증권형 토큰](23-wallet-sto.png)

증권형 토큰 및 토큰화된 증권 관리:
- **Vault Security Token (vVAULT)**: 플랫폼의 기본 증권형 토큰
- **토큰 상태**: 현재 상태 표시 (대기중 = Pending, 활성 = Active)
- **총 발행량**: 100,000,000.0 vVAULT 토큰
- **보유 잔액**: 보유한 vVAULT 토큰 수량
- **토큰 세부정보**: 토큰 경제학 및 스마트 계약 정보 확인

**vVAULT 토큰 정보:**
- 토큰화 수수료 지불에 사용
- 자산 토큰화 거래에 필수
- ETH를 스왑하여 획득 가능

#### 스왑 탭

![토큰 스왑](24-wallet-swap.png)

ETH와 vVault 토큰 간 교환:

**기능:**
- **스왑 방향 전환**:
  - ETH → vVault: ETH를 vVault 토큰으로 변환
  - vVault → ETH: vVault 토큰을 ETH로 변환
- **교환 비율**: 1 ETH = 1000 vVault 토큰
- **금액 입력**: 스왑할 금액 입력
- **예상 수령액**: 받을 금액 표시
- **잔액 표시**: ETH 및 vVault 잔액 모두 표시
- **새로고침 버튼**: 현재 잔액 업데이트

**스왑 방법:**
1. 스왑 방향 선택 (ETH → vVault 또는 vVault → ETH)
2. 스왑할 금액 입력
3. 예상 수령액 확인
4. **"스왑 실행"** 클릭
5. 거래 확인

**중요 참고사항:**
> vVault 토큰은 자산 토큰화 수수료에 사용됩니다. 실물 자산을 토큰화하기 전에 vVault 토큰을 확보하세요.

#### 자산토큰화 탭

![자산 토큰화](25-wallet-tokenization.png)

실물 자산을 블록체인 NFT로 변환:

**자산 토큰화란?**
토큰화는 물리적 자산(부동산, 회사 지분, 예술품 등)을 블록체인 기반 NFT로 변환하여 디지털 소유권 증명서를 생성합니다:
- 블록체인에서 거래 가능
- 분할 가능 (분할 소유권 가능)
- 투명 (소유권 이력 기록)
- 안전 (블록체인 보호)

**토큰화 프로세스:**
1. **"자산 토큰화"** 또는 **"첫 자산 토큰화하기"** 클릭
2. 자산 세부정보 제공 (이름, 설명, 평가액)
3. 관련 문서 업로드
4. 토큰화 수수료 지불 (100.0 vVAULT)
5. 블록체인에 자산이 NFT로 기록
6. 디지털 소유권 증명서 수령

**토큰화 수수료:**
- **비용**: 자산당 100.0 vVAULT 토큰
- **목적**: 블록체인 거래 비용 및 플랫폼 서비스 제공
- **요구사항**: 토큰화 전 충분한 vVault 잔액 보유 필수

**토큰화된 자산 관리:**
- 이 탭에서 모든 토큰화된 자산 확인
- 자산 가치 및 소유권 이력 추적
- 다른 사용자에게 소유권 이전
- 블록체인 거래 기록 확인

**빈 상태:**
아직 토큰화된 자산이 없을 때:
- "토큰화된 자산이 없습니다" 메시지
- 첫 실물 자산 토큰화 안내
- 토큰화 프로세스 시작 버튼

## 그룹 관리

**"그룹 관리"**를 클릭하여 협업 그룹을 생성하고 관리하세요.

![그룹 관리 페이지](15-groups-management.png)

### 그룹 탭

**내 그룹**:
- 생성한 모든 그룹 확인
- 그룹 멤버 및 권한 관리

**받은 초대**:
- 다른 사용자로부터 받은 그룹 초대 확인
- 초대 수락 또는 거절

### 새 그룹 만들기

**"새 그룹 만들기"** 버튼을 클릭하세요.

![그룹 생성 대화상자](20-create-group-dialog.png)

**그룹 생성 양식:**
1. **그룹 이름*** - 필수
   - 그룹의 설명적인 이름 입력
   - 예: "마케팅 팀" 또는 "프로젝트 알파"

2. **설명** - 선택사항
   - 그룹의 목적에 대한 추가 설명 제공
   - 멤버들이 그룹의 기능을 이해하는 데 도움

3. **"만들기"**를 클릭하여 그룹 완성

## 보안 모범 사례

### 지갑 보안

1. **PIN 기억하기**:
   - 오프라인으로 안전하게 적어두세요
   - 누구와도 공유하지 마세요
   - 디지털 형식으로 저장하지 마세요

2. **지갑 백업**:
   - 조각 A가 브라우저에 남아있는지 확인
   - 조각 B 접근을 위해 PIN을 안전하게 보관
   - 플랫폼이 조각 C를 유지

3. **2-of-3 복구**:
   - 지갑 복구를 위해 2개의 조각이 필요
   - PIN을 잃어도 지갑을 잃는 것은 아님
   - 브라우저 데이터 보존이 복구에 도움

### 금고 보안

1. **별도 비밀번호 사용**:
   - 계정, 금고, PIN에 다른 비밀번호 사용
   - 서비스 간 비밀번호 재사용 금지

2. **정기적인 비밀번호 변경**:
   - 금고 비밀번호를 주기적으로 업데이트
   - 강력하고 복잡한 비밀번호 사용

3. **활동 로그 모니터링**:
   - 활동 로그를 정기적으로 확인
   - 의심스러운 활동 즉시 보고

### 데이터 보안

1. **클라이언트 측 암호화**:
   - 모든 파일이 업로드 전에 암호화됨
   - 암호화 키가 암호화되지 않은 상태로 기기를 떠나지 않음

2. **제로 트러스트 아키텍처**:
   - 플랫폼은 기본적으로 어떤 프로세스도 신뢰하지 않음
   - 모든 작업에 검증 필요

3. **블록체인 투명성**:
   - 주요 활동이 블록체인에 기록됨
   - 모든 거래의 변경 불가능한 감사 추적

## 문제 해결

### 로그인할 수 없나요?

**확인사항:**
- 이메일 주소가 정확한지
- 비밀번호가 정확한지 (대소문자 구분)
- 계정이 인증되었는지

**해결방법:**
- 비밀번호 재설정 기능 사용
- 인증 링크가 있는지 이메일 확인
- 문제가 지속되면 지원팀에 문의

### PIN을 잊어버렸나요?

**브라우저 데이터가 보존된 경우:**
1. PIN 변경 페이지에서 "PIN을 잊으셨나요?" 클릭
2. 시스템이 조각 A (브라우저) + 조각 C (서버) 사용
3. 새 PIN 설정

**브라우저 데이터가 손실된 경우:**
- 계정 세부정보로 지원팀에 문의
- 복구 프로세스에 시간이 더 걸릴 수 있음
- 추가 검증이 필요할 수 있음

### 금고 비밀번호를 잊어버렸나요?

1. 계정 설정 → 비밀번호로 이동
2. 비밀번호 복구 링크 클릭
3. 검증 프로세스 따르기
4. 새 금고 비밀번호 설정

### 파일이 업로드되지 않나요?

**확인사항:**
- 스토리지 할당량 초과 여부
- 파일 크기가 제한 내에 있는지
- 네트워크 연결이 안정적인지
- 금고가 잠금 해제되었는지

**해결방법:**
- 스토리지 공간 확보
- 큰 파일 분할
- 업로드 재시도
- 문제가 지속되면 지원팀에 문의

### 지갑 잔액이 표시되지 않나요?

**개발 환경:**
- Hardhat Network가 실행 중인지 확인
- "새로고침" 버튼 클릭
- "테스트 ETH 받기"를 사용하여 테스트 자금 받기

**확인사항:**
- 네트워크 연결
- 블록체인 노드 상태
- 지갑 동기화
